Индивидуальный разработчик (автономно)[[Индивидуальный разработчик (автономно)]]
Автономный индивидуальный разработчик не обменивается патчами с другими людьми и работает самостоятельно в одном репозитории, используя следующие команды.

git-init[1] для создания нового репозитория.

git-show-branch[1], чтобы увидеть, где вы находитесь.

git-log[1], чтобы посмотреть, что произошло.

git-checkout[1] и git-branch[1] для переключения ветвей.

git-добавьте[1] для управления индексным файлом.

git-diff[1] и git-status[1], чтобы увидеть, что вы делаете в середине.

git-commit[1] для продвижения текущей ветки.

git-reset[1] и git-checkout[1] (с параметрами pathname) для отмены изменений.

git-merge[1] для объединения локальных ветвей.

git-перебазируйте[1] для поддержания ветвей темы.

git-тег[1], чтобы отметить известную точку.

Примеры
Используйте tarball в качестве отправной точки для нового репозитория.
$ tar zxf frotz.tar.gz
$ cd frotz
$ git инициализация
$ git add . (1)
$ git commit -m "импорт исходного дерева frotz".
$ git tag v2.43 (2)
добавьте все в текущий каталог.
